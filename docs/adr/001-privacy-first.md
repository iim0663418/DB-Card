# ADR-001: 隱私優先設計原則

## 狀態
已接受 (2026-01-17)

## 背景
NFC 數位名片系統需要在便利性與隱私保護之間取得平衡。傳統名片系統通常將資料集中儲存於伺服器，存在隱私洩露風險。

## 決策
採用「隱私優先」設計原則：
1. **最小化資料收集**: 僅收集必要的名片資訊
2. **加密儲存**: 所有名片資料使用信封加密儲存於 D1 Database
3. **授權會話**: 透過 ReadSession 機制控制資料存取
4. **IP 匿名化**: 審計日誌中的 IP 地址僅保留前 3 段
5. **無追蹤**: 不使用第三方分析工具追蹤使用者行為

## 後果

### 正面
- 符合 GDPR 和 Taiwan PDPA 隱私法規
- 使用者信任度提升
- 減少資料洩露風險

### 負面
- 無法進行詳細的使用者行為分析
- 審計日誌的追蹤能力受限
